# "게임은 안 보이는 곳을 그리지 않는다" — 콘텐츠 기획

## 콘텐츠 개요

| 항목 | 내용 |
|------|------|
| **제목 (안)** | "게임은 안 보이는 곳을 그리지 않는다" |
| **핵심 메시지** | 게임은 플레이어가 볼 수 없는 것을 그리지 않는다. "안 보이는 것"을 판단하는 방법은 하나가 아니며, 여러 기법이 겹겹이 쌓여 게임이 실시간으로 동작하게 만든다. |
| **타겟 시청자** | 코딩을 모르는 일반 게이머 / 게임에 관심 있는 비개발자 |
| **톤** | 호기심 유발 → 직관적 시각화 → 체감 → 다음 주제 기대감 |
| **깊이 수준** | 각 기법의 존재와 효과를 시각적으로 보여주고 직관적으로 이해시킨다. 내부 구현 원리(AABB, 평면 테스트 등)는 깊이 다루지 않는다. |
| **진행 방식** | 최적화가 없는 상태에서 시작해, 컬링 기법을 하나씩 적용하며 FPS가 개선되는 과정을 보여준다. 기법을 "소개"하는 것이 아니라 "적용해서 문제를 해결"하는 흐름으로 진행한다. |

---

## 씬 구성

### 씬 컨셉: 미니 도시

**분위기:** 건물, 가로등, 나무, 소품 등이 배치된 작은 도시/마을

**이 씬을 선택한 이유:**
- 게임에서 가장 흔한 배경이라 비개발자도 즉시 공감
- 건물 → Occlusion Culling 시연에 자연스러움 (건물 뒤에 숨은 오브젝트)
- 건물 벽면이 크고 평평 → Back-face Culling 효과가 눈에 잘 보임
- 가까운 건물/먼 건물 → LOD 시연의 자연스러운 원근감
- "이 도시가 전부 그려지고 있는 게 아니었다고?" 하는 반전 가능

**에셋:**
- 외부 glTF 모델 로딩 (폴리곤 수가 많은 모델로 최적화 효과를 극적으로 보여줌)
- 무료 에셋 활용 (건물, 가로등, 나무, 자동차 등)
- 소수의 모델을 크기/회전을 바꿔 반복 배치하여 밀도 있는 도시 장면 구성

## 장면 구성

### 파트 1: 문제 제시 — "왜 이렇게 느리지?"

**목표:** 최적화 없이 모든 것을 그리면 어떻게 되는지 보여준다.

**화면 연출:**
- 건물, 가로등, 나무, 자동차 등이 빼곡한 미니 도시 장면
- 모든 컬링 OFF 상태
- FPS 카운터가 한 자릿수 또는 10대로 떨어져 있음. 화면이 뻑뻑하게 움직임
- 드로우콜 수, 삼각형 수가 엄청나게 높은 것을 통계 오버레이로 보여줌
- 나레이션: "이 도시에는 건물, 나무, 자동차 등 수천 개의 오브젝트가 있습니다. 전부 다 그리고 있습니다. 보시다시피, 게임이 되지 않습니다."
- **핵심 질문 던지기:** "어떻게 하면 이걸 부드럽게 돌릴 수 있을까요?"

**전달 메시지:**
- 모든 것을 그대로 그리면 게임은 동작할 수 없다
- 이제부터 이 문제를 하나씩 해결해 나간다

---

### 파트 2: 첫 번째 개선 — "화면에 없으면 안 그린다" (Frustum Culling)

**목표:** 카메라 시야 밖을 그리지 않는 것만으로 큰 개선이 된다는 것을 보여준다.

**화면 연출:**
- 현재 상태 확인: FPS 낮음, 도시의 모든 오브젝트를 전부 그리는 중
- God View로 전환 → 도시를 위에서 내려다보며 카메라의 프러스텀(시야 영역)을 반투명 와이어프레임으로 표시
- 나레이션: "카메라 앞에 보이지 않는 피라미드가 있습니다. 이 안에 있는 것만 그리면 어떻게 될까요?"
- **Frustum Culling ON** → 프러스텀 밖의 건물, 나무 등이 회색/투명으로 변하며 사라짐
- FPS 즉시 상승. 통계: "수천 개 중 수백 개만 그리는 중"
- 플레이어 시점으로 돌아와서 도시를 돌아다니며 카메라를 회전 → 시야 밖의 건물이 걸러지는 것을 확인
- FOV 슬라이더 조절 → 시야가 넓어지면 더 많이 그림, 좁아지면 더 적게 그림
- Far Plane 슬라이더 조절 → 먼 거리를 잘라내면 더 줄어듦
- 나레이션: "옛날 게임에 안개가 많았던 이유, 이제 아시겠죠? 먼 거리를 잘라내서 그릴 양을 줄인 겁니다."

**전달 메시지:**
- 카메라 밖을 안 그리는 것만으로 대부분의 오브젝트를 절약할 수 있다
- 하지만 아직 충분하지 않다 — FPS가 완전히 회복되지는 않음

---

### 파트 3: 두 번째 개선 — "뒷면은 볼 일이 없다" (Back-face Culling)

**목표:** 오브젝트의 뒷면을 그리지 않으면 삼각형 수가 크게 줄어든다는 것을 보여준다.

**화면 연출:**
- 현재 상태 확인: Frustum Culling으로 개선됐지만, 삼각형 수가 여전히 많음
- 건물 하나를 클로즈업. 건물의 여러 면 중 카메라에서 보이지 않는 뒷면이 있음
- 뒷면을 빨간색으로 하이라이트 → "이 면들은 어차피 보이지 않습니다"
- 카메라를 건물 주위로 돌리면 어떤 면이 앞/뒤인지 실시간으로 바뀜
- 나레이션: "건물에는 항상 카메라를 등진 면이 있습니다. 안 보이는 면을 그리는 건 낭비입니다."
- **Back-face Culling ON** → 삼각형 수 카운터가 거의 절반으로 줄어듦
- FPS 추가 상승
- 나레이션: "면의 절반을 절약했습니다."

**전달 메시지:**
- 모든 물체의 뒷면을 건너뛰면 삼각형 수가 크게 줄어든다
- 여전히 더 개선할 수 있다

---

### 파트 4: 세 번째 개선 — "벽 뒤에 있으면 몰라도 된다" (Occlusion Culling)

**목표:** 다른 오브젝트에 가려진 것까지 걸러내면 또 한 번 큰 개선이 된다는 것을 보여준다.

**화면 연출:**
- 도시의 큰 건물 앞에 카메라를 위치 → 건물 뒤 골목에 나무, 자동차, 다른 건물들이 숨어 있음
- 플레이어 시점: 앞의 건물만 보임. 하지만 드로우콜이 여전히 높음
- God View 전환: 건물 뒤에 오브젝트가 잔뜩 있는 것이 보임. 프러스텀 안이므로 전부 그리고 있음
- 나레이션: "이 건물 뒤에 뭐가 있는지 플레이어는 모릅니다. 하지만 게임은 전부 그리고 있습니다. 이것도 걸러낼 수 있다면?"
- **Occlusion Culling ON** → 건물 뒤 오브젝트가 걸러짐. 드로우콜 급감
- FPS 추가 상승
- 나레이션: "시야 안에 있어도, 건물에 가려져 있으면 그리지 않습니다."

**전달 메시지:**
- Frustum Culling이 놓치는 부분을 Occlusion Culling이 잡아낸다
- 실내/도시처럼 가리는 것이 많은 장면에서 특히 효과적

---

### 파트 5: 네 번째 개선 — "멀리 있는 건 대충 그려도 모른다" (LOD / Distance Culling)

**목표:** 거리에 따라 디테일을 줄이거나 아예 안 그리면 추가 개선이 된다는 것을 보여준다.

**화면 연출:**
- 도시 끝 멀리 보이는 건물을 줌인 → 화면에서 몇 픽셀 크기밖에 안 됨
- 나레이션: "저 멀리 있는 건물은 화면에서 겨우 몇 픽셀입니다. 이걸 수천 개의 삼각형으로 정밀하게 그려봤자 플레이어는 알아채지 못합니다."
- **LOD / Distance Culling ON**
- 가까운 건물: 디테일 높은 모델 유지
- 중간 거리 건물: 단순화된 모델
- 먼 거리 건물/소품: 아예 안 그리거나 극도로 단순화
- 삼각형 수 카운터 추가 감소, FPS 추가 상승
- 도시를 걸어다니며 건물에 다가가면 디테일이 올라가고, 멀어지면 내려가는 것을 보여줌

**전달 메시지:**
- "보이긴 보이지만 의미 없는 것"도 최적화 대상이다
- 이제 FPS가 처음과는 비교할 수 없을 만큼 좋아졌다

---

### 파트 6: 결과 비교 — "처음과 지금"

**목표:** 처음의 뻑뻑한 장면과 최적화 후의 부드러운 장면을 극적으로 대비시킨다.

**화면 연출:**
- 화면을 분할하거나 빠르게 전환:
  - **Before:** 모든 컬링 OFF — FPS 한 자릿수, 도시의 모든 오브젝트를 전부 그림
  - **After:** 모든 컬링 ON — FPS 60, 실제로 그리는 건 수백 개
- 각 기법이 얼마나 기여했는지 단계별 요약:
  - Frustum Culling: "카메라 밖을 걸러냄"
  - Back-face Culling: "뒷면을 걸러냄"
  - Occlusion Culling: "가려진 것을 걸러냄"
  - LOD / Distance Culling: "먼 것을 단순화"
- 나레이션: "안 보이는 곳을 안 그린다. 이 단순한 원칙을 여러 겹으로 적용한 결과, 수천 개의 건물과 나무가 있는 도시가 60fps로 돌아갑니다."

**전달 메시지:**
- 하나의 기법만으로는 충분하지 않다 — 여러 기법이 겹쳐서 작동한다
- 게이머가 체감하는 부드러운 플레이 뒤에는 이런 최적화들이 숨어 있다

---

### 파트 7: 마무리 — "그래도 완벽하진 않다"

**목표:** 여운을 남기고 다음 콘텐츠에 대한 기대감을 만든다.

**화면 연출:**
- 컬링이 실패하는 엣지 케이스를 한두 개 가볍게 보여줌
  - 예: 프러스텀 경계에서 오브젝트가 깜빡이는 현상
  - 예: LOD 전환이 눈에 보이는 "팝인" 현상
- 나레이션: "완벽한 기술은 없습니다. 게임 개발자들은 이 한계를 극복하기 위해 끊임없이 새로운 방법을 찾고 있습니다."
- 다음 영상 주제 예고 (예: 그림자, 라이팅 등)

## 쇼케이스 앱 인터랙션

### 뷰 모드
| 모드 | 설명 | 전환 방법 |
|------|------|-----------|
| **플레이어 뷰** | 일반 1인칭 카메라 시점 | 기본 모드 |
| **God View** | 위에서 내려다보는 전지적 시점. 프러스텀 와이어프레임 표시 | 버튼 또는 단축키 |

### ImGui 컨트롤 패널
| 컨트롤 | 타입 | 용도 |
|---------|------|------|
| Frustum Culling | 토글 | 프러스텀 컬링 ON/OFF |
| Back-face Culling | 토글 | 백페이스 컬링 ON/OFF |
| Occlusion Culling | 토글 | 오클루전 컬링 ON/OFF |
| LOD / Distance Culling | 토글 | LOD 및 거리 컬링 ON/OFF |
| FOV | 슬라이더 (30°~120°) | 시야각 조절 |
| Far Plane | 슬라이더 | 원거리 절단면 조절 |
| Object Count | 슬라이더 (100~10,000) | 오브젝트 수 조절 |
| Show Frustum | 토글 | 프러스텀 와이어프레임 표시 |
| Show Back-faces | 토글 | 뒷면 하이라이트 표시 |
| Highlight Culled | 토글 | 컬링 상태 색상 표시 |

### 실시간 통계 (ImGui 오버레이)
- FPS
- 총 오브젝트 수 / 실제로 그리는 오브젝트 수
- 총 삼각형 수 / 실제로 그리는 삼각형 수
- 드로우콜 수
